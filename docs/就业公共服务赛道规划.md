# BossWork · 就业公共服务、职业指导与职业培训数据分析服务

> **赛道定位**：探索「人工智能 + 就业服务」应用场景，推动「大数据 + 铁脚板」服务模式，为政策制定与求职者服务提供数据支撑。  
> **项目代号**：BossWork  
> **文档版本**：v1.0  
> **更新日期**：2026-06-29

---

## 一、赛道命题解读

### 1.1 命题要求摘要

| 维度 | 命题要求 |
|------|----------|
| 服务模式 | 「人工智能 + 就业服务」「大数据 + 铁脚板」 |
| 数据输入 | 企业招聘需求、技术专利、政策文件、宏观经济等 |
| 分析能力 | 预测中短期重点产业/区域技能需求趋势 |
| 核心输出 | 供需匹配、职业发展路径、技能缺口分析 |
| 服务功能 | 就业信息推送、个性化职业规划、技能提升建议 |
| 服务对象 | 求职者（C 端）、政策制定方（G 端）、培训/院校（B 端） |

### 1.2 与本项目的契合点

BossWork 从 Boss 直聘招聘大数据采集出发，已构建 **数据采集 → 技能分析 → AI 人岗匹配 → 智能求职沟通** 的技术链路，与赛道命题中的「招聘需求分析」「供需匹配」「个性化职业指导」「AI 就业服务」高度吻合，具备以最小增量完成赛道升级的基础。

**一句话定位**：

> 基于招聘市场大数据与 AI Agent 的智能化就业公共服务原型系统，实现市场需求洞察、个人技能缺口识别、岗位智能推荐与职业指导一体化服务。

## 二、目前系统情况

### 2.1 系统架构（已实现）

```
┌─────────────────────────────────────────────────────────────┐
│                    app.py / chat_agent.py                     │
│              CLI 交互入口 · 单次任务 · 浏览器执行              │
└─────────────────────────┬───────────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────────┐
│              BossWorkOrchestrator（Triage Agent）             │
│         handoff ──┬──────────┬──────────┐                   │
└───────────────────┼──────────┼──────────┼───────────────────┘
                    ▼          ▼          ▼
            JobMatcher   JobSeekerAgent   OutreachPlanner
            岗位筛选      求职者回复 HR      主动联系策略
                    │          │          │
                    └──────────┴──────────┘
                               │ function_tool
┌──────────────────────────────▼──────────────────────────────┐
│  BossWorkContext（profile / jobs / pending_actions）         │
└──────────────────────────────┬──────────────────────────────┘
         │                                      │
         ▼                                      ▼
  services/jobs.py                    runtime/browser.py
  岗位加载/筛选/匹配                   Playwright 登录态发消息
  scraper.py + skill_analyzer.py      chat_client.py + llm_reply.py
  数据采集 · 技能提取 · 频次统计       HR 实时沟通 · LLM 回复生成
```

### 2.2 已实现功能清单

#### （1）招聘大数据采集层

| 模块 | 文件 | 能力 |
|------|------|------|
| 浏览器采集 | `scraper.py`、`anti_bot.py` | Playwright + 本机 Chrome，绕过 Boss 反爬（security-check、__zp_stoken__ 等） |
| 采集 CLI | `main.py` | 按城市/关键词批量抓取，支持 ui / api 双模式 |
| 数据字段 | `output/boss_jobs_*.json\|csv` | 岗位名、薪资、经验、学历、城市、公司、行业、技能标签、JD、HR 信息等 20+ 字段 |
| 覆盖范围 | `config.py` | 预置 20 个计算机类关键词，支持北京/上海/广州/深圳等 10 城 |

**当前数据样例**（2026-06-29 采集）：

- 已具备广州等地 Java 开发/实习岗位 CSV/JSON 输出
- 每条记录含 `extracted_skills`、`company_industry`、`target_city` 等结构化字段

#### （2）技能数据分析层

| 模块 | 文件 | 能力 |
|------|------|------|
| 技能提取 | `skill_analyzer.py` | 从岗位标签、JD 文本中提取技能关键词 |
| 频次统计 | `output/skill_summary.json` | 统计技能在岗位集中的出现频次 |
| 关键词库 | `config.py` → `SKILL_KEYWORDS` | 预置常见 IT 技能词典 |

#### （3）AI 就业服务层（OpenAI Agents SDK）

| Agent | 职责 | 核心工具 |
|-------|------|----------|
| **BossWorkOrchestrator** | 意图理解、任务路由 | handoffs |
| **JobMatcher** | 按城市/技能筛选推荐岗位 | `filter_job_listings`、`get_job_by_id` |
| **JobSeekerAgent** | 以求职者身份回复 HR | `reply_to_hr_message`、`queue_browser_reply` |
| **OutreachPlanner** | 制定打招呼策略与话术 | `queue_browser_greet` |

#### （4）求职者画像与匹配

| 模块 | 文件 | 能力 |
|------|------|------|
| 个人资料 | `user_profile.json` | 姓名、技能、项目、期望薪资、城市偏好、回复规则等 |
| 匹配算法 | `bosswork/services/jobs.py` | 城市过滤 + 技能重叠打分排序 |
| Prompt 注入 | `bosswork/services/profile.py` | 将画像注入 Agent 与 LLM 系统提示 |

#### （5）「铁脚板」落地执行层

| 模块 | 文件 | 能力 |
|------|------|------|
| HR 实时沟通 | `chat_agent.py`、`chat_client.py` | 轮询聊天 API、读取历史消息 |
| LLM 回复 | `llm_reply.py` | 结合 profile + 岗位 + 对话历史生成求职者回复 |
| 浏览器自动化 | `bosswork/runtime/browser.py` | 排队执行打招呼/发消息（Human-in-the-loop 确认） |

### 2.3 技术栈

| 类别 | 技术 |
|------|------|
| 语言 | Python 3.11+ |
| 采集 | Playwright、本机 Chrome |
| AI 框架 | OpenAI Agents SDK |
| LLM | OpenAI 兼容 API（GPT / DeepSeek / 通义等） |
| 数据格式 | JSON、CSV |
| 交互 | CLI（命令行） |

### 2.4 当前系统边界

- **用户形态**：单人求职者，静态 JSON 配置
- **行业范围**：以计算机/互联网类岗位为主
- **数据来源**：Boss 直聘单平台
- **分析深度**：技能频次统计，无时序趋势与预测
- **交互形态**：命令行，无 Web 可视化界面
- **服务闭环**：无推荐效果追踪与服务评估指标

### 3.3 竞争优势（可写入方案）

1. **端到端闭环**：从市场数据采集到 AI 推荐、再到真实 HR 沟通，链路完整且可演示  
2. **Agent 架构先进**：基于 OpenAI Agents SDK，Handoffs + Tools 可扩展  
3. **真实数据驱动**：非 Mock，来自 Boss 直聘真实招聘市场  
4. **Human-in-the-loop**：半自动沟通设计，兼顾效率与合规  

### 3.4 主要短板（需在方案中说明应对）

1. 数据源单一，缺政策/宏观/专利等多维数据  
2. 分析偏静态，缺趋势预测与结构化报告  
3. 无 Web 界面，比赛演示体验不足  
4. 行业聚焦 IT，公共就业服务覆盖面需扩展 narrative

## 四、后续开发计划

### 4.1 总体目标

将 BossWork 从「个人求职 Agent 工具」升级为面向赛道的 **「招聘大数据分析 + AI 职业指导服务平台」**，在 4–6 周内交付可演示 MVP。

### 4.2 分期计划

#### Phase 1：分析能力补全（第 1–2 周）

**目标**：补齐赛道核心分析能力，形成可输出的结构化报告。

| 任务 | 产出 | 优先级 |
|------|------|--------|
| 历史数据归档机制 | 按日期存储采集快照，支持时序对比 | P0 |
| 技能趋势分析模块 | 技能升降趋势、区域/行业对比报告 | P0 |
| 技能缺口分析 | 输入 profile → 输出「已具备 / 市场热门 / 待补齐」 | P0 |
| 供需匹配报告 | 城市 × 技能维度的岗位供给量统计 | P1 |
| 扩展行业关键词 | 增加 2–3 个非 IT 行业 demo 采集配置 | P2 |

**新增模块建议**：

```
bosswork/
├── analytics/
│   ├── trends.py        # 技能趋势分析
│   ├── gap.py           # 个人技能缺口
│   └── supply_demand.py # 供需统计
└── data/
    └── snapshots/       # 按日落盘的历史数据
```

#### Phase 2：职业指导与培训建议（第 2–3 周）

**目标**：实现「个性化职业规划 + 技能提升建议」。

| 任务 | 产出 | 优先级 |
|------|------|--------|
| 职业发展路径 Agent | 基于目标岗位生成 3 阶段成长路径 | P0 |
| 培训建议引擎 | 技能缺口 → 课程/学习资源映射 | P0 |
| 职业规划报告生成 | LLM 生成结构化 PDF/Markdown 报告 | P1 |
| 课程资源库（Mock） | 内置常见 IT 技能课程目录 JSON | P1 |

**新增 Agent 建议**：

| Agent | 职责 |
|-------|------|
| **CareerPlanner** | 职业路径规划、中长期目标拆解 |
| **SkillCoach** | 技能缺口解读、培训建议、学习优先级 |

#### Phase 3：多源数据与政策支撑（第 3–4 周）

**目标**：满足赛道「多源数据 + 政策支撑」叙事，不要求全部真实对接。

| 任务 | 产出 | 优先级 |
|------|------|--------|
| 政策文件知识库 | 收录就业/产业/培训相关政策摘要（10–20 条） | P1 |
| 宏观数据 Mock/API | 接入公开统计数据或内置 demo 数据集 | P1 |
| 专利/产业趋势（可选） | 简化版行业热度指数 | P2 |
| 数据融合层 | 统一数据接入接口，支持扩展 | P1 |

#### Phase 4：Web 可视化与演示（第 4–5 周）

**目标**：交付比赛 Demo 级 Web 产品。

| 页面 | 功能 |
|------|------|
| **市场洞察看板** | 技能热力图、区域供需、趋势折线图 |
| **个人职业中心** | 画像编辑、缺口雷达图、推荐岗位列表 |

**技术选型建议**：

## 六、应用场景设计（写入方案）

### 6.1 求职者（C 端）

**场景**：应届毕业生小李，计算机专业，希望在广州找 Java 后端工作。

1. 系统采集广州 Java 岗位大数据，分析当前最热门技能（Spring Cloud、Redis、Kafka…）
2. 对比小李 profile，生成技能缺口报告（缺 Kafka、微服务实战经验）
3. CareerPlanner 给出 6 个月成长路径：补 Kafka → 做微服务项目 → 投中级岗位
4. SkillCoach 推荐相关培训课程与学习资源
5. JobMatcher 推荐 Top 10 匹配岗位
6. JobSeekerAgent 协助与 HR 智能沟通

### 6.2 政策制定（G 端）

**场景**：人社部门需要了解某市 IT 技能需求变化，为培训补贴政策提供依据。

1. 市场看板展示近 3 个月技能需求趋势（如 AI/大模型技能需求上升 40%）
2. 区域对比：广州 vs 深圳 Java 岗位供给与薪资差异
3. 结合政策库，提示现有培训目录与市场需求错配项
4. 输出《区域技能需求分析简报》供决策参考

### 6.3 院校/培训（B 端）

**场景**：某职业院校优化 Java 专业课程设置。

1. 分析近 6 个月企业招聘 JD 中技能要求分布
2. 对比现有课程大纲，识别覆盖不足的技能点
3. 生成「课程优化建议」：增加 Redis、消息队列、云原生等模块

## 七、风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| Boss 反爬升级 | 数据采集中断 | 已有 anti_bot 模块；准备备用数据源或历史快照 |
| 多源数据难获取 | 方案完整性 | 政策/宏观用公开数据 + Mock；方案中说明扩展路径 |
| 开发周期紧 | 无法全部实现 | 优先 P0 项；Web 可用 Streamlit 快速出 Demo |
| 比赛演示依赖网络 | 现场失败 | 预录视频 + 本地离线数据快照 |
| 合规风险 | 评审质疑 | 强调学习研究用途、频率控制、Human-in-the-loop |

## 八、总结

| 维度 | 现状 | 目标 |
|------|------|------|
| 定位 | 个人求职 Agent 工具 | 就业公共服务数据分析平台 |
| 数据 | Boss 单平台 | Boss + 政策 + 宏观 + 培训 |
| 分析 | 技能频次 | 趋势 + 缺口 + 供需 + 路径 |
| 服务 | 岗位推荐 + HR 对话 | 全流程职业指导 + 培训建议 |
| 界面 | CLI | Web 可视化 Demo |
| 闭环 | 无 | 推送 + 行为追踪 + 效果评估 |

BossWork 已具备赛道所需的核心骨架（招聘大数据 + AI 就业服务 + 人岗匹配 + 落地执行），后续开发重点应放在 **分析层补全、职业指导深化、Web 演示交付** 三个方向，以最小成本完成从「工具」到「平台」的赛道适配升级。
